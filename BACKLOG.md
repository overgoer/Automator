# Automator — Бэклог

## 📋 Контекст

**Automator** — ИИ-контент креатор для Telegram канала @eddytester.

### Принципы
1. **СКОРОСТЬ** — быстро дожить до прода, улучшать потом
2. **НАДЁЖНОСТЬ** — работает как Toyota Land Cruiser
3. **ПРОСТОТА** — минимум зависимостей
4. **ГИБКОСТЬ** — адаптируемся по ходу

> **Философия**: Костыль который работает, лучше идеала который не запущен.

---

## ✅ Реализовано (v0.2.0)

- [x] Бот @mrgriffbot — админ в @eddytester
- [x] Чтение постов (aiogram long polling)
- [x] DeepSeek AI интеграция
- [x] Генерация постов в стиле канала
- [x] Отправка поста пользователю (@edtext)
- [x] Логирование
- [x] Обработка ошибок

### Как работает v0.2.0

```
Пост в канале → Бот читает → ИИ генерирует → Отправка тебе
```

**Ограничение**: Работает в реальном времени. Бот должен быть запущен постоянно.

## 🔥 Бэклог (приоритеты)

### Критические

#### 1. Развёртывание на сервере
**Зачем**: Бот должен работать 24/7.

**Варианты**:
- [ ] Твой сервер (Docker или systemd)
- [ ] VPS (DigitalOcean, Linode, Hetzner)
- [ ] PaaS (Railway, Render, Fly.io)

**Оценка**: 2-4 часа

---

### Высокий приоритет

#### 2. Чтение истории канала
**Зачем**: Обрабатывать существующие посты, не только новые.

**Варианты**:
- [ ] Получить API_ID/HASH с my.telegram.org
- [ ] Использовать Telethon с user сессией
- [ ] Скрапинг через tdlib

**Оценка**: 2-8 часов

---

#### 3. Email уведомления (резерв)
**Зачем**: Если Telegram недоступен.

**Задачи**:
- [ ] SMTP интеграция
- [ ] Fallback логика

**Оценка**: 2 часа

---

#### 4. Режим черновика
**Зачем**: Проверка перед публикацией.

**Задачи**:
- [ ] Флаг DRAFT_MODE
- [ ] Команда /publish

**Оценка**: 3 часа
