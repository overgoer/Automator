# Automator — Бэклог и дорожная карта

## 📋 Контекст проекта

**Automator** — ИИ-контент креатор для Telegram канала @eddytester.

### Миссия
Автоматизировать создание контента для QA-канала: анализ последних постов → генерация нового поста через ИИ → отправка на публикацию.

### Целевое состояние
- Работает автономно 24/7
- Генерирует контент в стиле канала
- Требует минимального вмешательства человека
- Надёжно как "Toyota Land Cruiser"

### 🔥 Принципы (в порядке приоритета)
1. **СКОРОСТЬ РЕАЛИЗАЦИИ** — быстро дожить до прода, улучшать потом
2. **НАДЁЖНОСТЬ** — работает без сбоев
3. **ПРОСТОТА** — минимум зависимостей, минимум сложности
4. **ГИБКОСТЬ** — адаптируемся по ходу, не застреваем в планировании

> **Философия**: Костыль который работает, лучше идеала который не запущен.

### ⚠️ Известные проблемы v0.1.0

**Проблема 1: GitHub Actions не подходит для постоянной работы**
- GitHub Actions — временные раннеры (5-10 мин), не сервер
- Сессия Telegram и SQLite не сохраняются между запусками
- **Статус**: Требуется решение (локальный запуск или сервер)

**Проблема 2: Отсутствуют TELEGRAM_API_ID/HASH**
- Нужны для Telethon сессии (второй аккаунт)
- **Статус**: Пользователь должен получить на my.telegram.org/apps

## 🎯 Бэклог (приоритеты)

### 🔥 КРИТИЧЕСКИЕ (блокеры прода)

#### 0. Решить проблему хостинга
**Зачем**: GitHub Actions не работает для постоянной работы.

**Варианты**:
- **A. Локальный запуск** — пользователь запускает руками раз в 2 дня
- **B. Сервер пользователя** — постоянный демон 24/7
- **C. GitHub Actions + внешнее хранилище** — сложно, небезопасно

**Задачи**:
- [ ] Выбрать вариант (рекомендуется B — сервер)
- [ ] Для A: инструкция по локальному запуску
- [ ] Для B: Dockerfile + инструкция по деплою
- [ ] Для C: отказаться от этой идеи

**Оценка**: 2-8 часов (зависит от варианта)

**Идея: Упростить архитектуру**

- Предложение: Использовать tdl или экспорт из Telegram Desktop
- Статус: Отклонено — нужен автоматический запуск

**Идея: GitHub Actions как сервер**
- Проблема: Думали что GA — это постоянный сервер
- Реальность: GA — временные раннеры (5-10 мин)
- Статус: Отклонено, требуется альтернатива

**Идея: tdl + JSON дамп**
- Суть: tdl скачивает посты в JSON → ИИ читает файл → бот отправляет
- Статус: В бэклоге как альтернатива Telethon
